# CuiSelect

`CuiSelect` 是对 Element Plus `ElSelect` 的标准封装，统一了选项数据格式，简化了常见的单选、搜索场景。

#code

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { CuiSelect } from '@eams-monorepo/vue-element-cui'
import type { CuiSelectOption } from '@eams-monorepo/vue-element-cui'

const selected = ref<string | null>(null)
const options: CuiSelectOption[] = [
  { label: '固定资产', value: 'fixed' },
  { label: '流动资产', value: 'current' },
  { label: '无形资产', value: 'intangible' },
]
</script>

<template>
  <CuiSelect v-model="selected" :options="options" placeholder="请选择资产类型" />
</template>
```

::api-table{title="Props" description="CuiSelect 配置项"}
| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `modelValue` | `any` | — | 当前选中值，支持 v-model（必填） |
| `options` | `CuiSelectOption[]` | `[]` | 选项列表 |
| `placeholder` | `string` | `'请选择'` | 占位文本 |
| `disabled` | `boolean` | `false` | 是否禁用 |
| `clearable` | `boolean` | `true` | 是否可清空 |
| `filterable` | `boolean` | `true` | 是否可过滤搜索 |
| `remote` | `boolean` | `false` | 是否开启远程搜索 |
| `remoteMethod` | `(query: string) => void` | — | 远程搜索方法 |
| `loading` | `boolean` | `false` | 是否显示加载中状态 |
::

::api-table{title="Emits" description="CuiSelect 事件"}
| 事件名 | 参数 | 说明 |
| --- | --- | --- |
| `update:modelValue` | `value: any` | 选中值变化时触发，用于 v-model 绑定 |
| `change` | `value: any` | 选中值变化时触发 |
::

::doc-note{title="与 ElSelect 的关系" tone="info"}
`CuiSelect` 是 `ElSelect` 的薄封装，核心差异在于通过 `options` prop 统一管理候选项，省去手写 `el-option` 列表的繁琐。底层仍使用 `ElSelect`，绝大多数 `ElSelect` 原有属性仍可透传。
::
