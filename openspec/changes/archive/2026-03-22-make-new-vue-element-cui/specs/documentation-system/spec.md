## 新增需求

> **重要说明：** 本规范已被 `improve-vue-element-cui-nuxt-docs-with-docus` 变更取代。
> 文档系统现在使用 Docus 框架而非基础 Nuxt Content。
> 请参考 `openspec/changes/improve-vue-element-cui-nuxt-docs-with-docus/` 获取最新的文档系统规范。
>
> 以下内容仅作为历史参考保留。

### 需求：Nuxt 3 项目结构

文档系统应当是一个位于 `packages/vue-element-cui-nuxt/` 的 Nuxt 3 项目，同时作为演练场和文档站点。

#### 场景：项目初始化

- **当** 开发者初始化 Nuxt 项目时
- **则** 项目应当包含 `pages/`、`content/` 和 `components/` 目录

#### 场景：开发服务器

- **当** 开发者运行 `pnpm dev` 时
- **则** Nuxt 开发服务器应当启动并支持热重载

### 需求：Nuxt Content 集成

文档系统应当使用 Nuxt Content 来实现基于 Markdown 的文档，并支持 MDC 语法。

#### 场景：内容目录

- **当** 开发者创建文档时
- **则** Markdown 文件应当放置在 `content/docs/` 目录中

#### 场景：MDC 语法

- **当** 文档包含组件演示时
- **则** 应当支持使用 MDC 语法直接在 Markdown 中嵌入 Vue 组件

### 需求：组件演练场

文档系统应当在专用页面上提供交互式组件演示。

#### 场景：组件演示页面

- **当** 用户导航到组件页面时
- **则** 页面应当显示带有实时代码的交互式组件示例

#### 场景：组件导入

- **当** 演练场使用库组件时
- **则** 应当使用 workspace 协议从 `vue-element-cui` 包导入

### 需求：文档结构

文档应当包含入门指南和各个组件的文档页面。

#### 场景：入门页面

- **当** 用户访问文档时
- **则** 应当在 `/docs/getting-started` 找到安装和快速入门指南

#### 场景：组件文档

- **当** 用户查看组件文档时
- **则** 每个组件应当在 `/docs/components/<component-name>` 有专用页面

### 需求：代码高亮

文档应当为代码示例提供语法高亮。

#### 场景：代码块高亮

- **当** 文档包含代码块时
- **则** Nuxt Content 应当为 Vue、TypeScript 和 Bash 提供语法高亮

#### 场景：主题配置

- **当** 代码块被渲染时
- **则** 应当使用 `github-dark` 主题以保持一致的外观

### 需求：导航

文档应当提供侧边栏导航以便于浏览。

#### 场景：侧边栏导航

- **当** 用户查看文档时
- **则** 侧边栏应当显示所有文档页面的层级导航

#### 场景：当前页面指示

- **当** 用户在某个文档页面上时
- **则** 侧边栏应当高亮显示当前页面
