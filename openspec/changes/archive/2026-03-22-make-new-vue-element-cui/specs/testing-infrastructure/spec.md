## 新增需求

### 需求：vitest 配置

测试基础设施必须使用 vitest 作为测试运行器，并使用 jsdom 环境进行组件测试。

#### 场景：测试框架设置

- **当**开发者运行测试时
- **则** vitest 必须在 jsdom 环境中执行测试

#### 场景：Vue 组件测试

- **当**开发者测试 Vue 组件时
- **则** @vue/test-utils 必须可用于组件挂载和交互

### 需求：测试覆盖率

测试基础设施必须跟踪代码覆盖率并生成覆盖率报告。

#### 场景：覆盖率配置

- **当**开发者运行带覆盖率的测试时
- **则** vitest 必须生成文本、JSON 和 HTML 格式的覆盖率报告

#### 场景：覆盖率阈值

- **当**测试完成时
- **则** P0 组件必须达到 ≥85% 覆盖率，P1 ≥75%，P2 ≥70%

### 需求：测试组织

测试必须组织在 `src/tests/` 目录中，并遵循清晰的命名约定。

#### 场景：测试文件命名

- **当**开发者创建测试文件时
- **则**文件必须命名为 `<component-name>.test.ts`

#### 场景：测试文件位置

- **当**开发者创建测试时
- **则**测试必须放置在 `src/tests/` 目录中

### 需求：测试结构

测试必须使用 describe/test 块，并具有清晰、描述性的名称。

#### 场景：测试组织

- **当**开发者编写测试时
- **则**必须使用 `describe('<ComponentName>', () => {})` 进行分组

#### 场景：测试命名

- **当**开发者编写单个测试时
- **则**必须使用 `test('should <behavior>', () => {})` 格式

### 需求：组件测试模式

测试必须覆盖 props、事件、插槽和组件行为。

#### 场景：Props 测试

- **当**组件接收 props 时
- **则**测试必须验证组件在不同 prop 值下正确渲染

#### 场景：事件测试

- **当**组件触发事件时
- **则**测试必须验证事件以正确的载荷触发

#### 场景：插槽测试

- **当**组件有插槽时
- **则**测试必须验证插槽内容正确渲染

### 需求：类型测试

测试必须验证 TypeScript 类型推断和类型安全。

#### 场景：泛型类型推断

- **当**组件使用泛型类型时
- **则**测试必须使用 `expectTypeOf` 验证类型推断正确工作

#### 场景：Props 类型检查

- **当**定义组件 props 时
- **则**测试必须验证 TypeScript 在编译时捕获无效的 prop 类型
