## ADDED Requirements

### Requirement: 文档内容目录必须按新的四级栏目结构重排

系统 SHALL 将文档内容目录重排为“快速开始 / 组件 / 规范 / 更新”的顶级结构，并以此作为正式内容入口。

#### Scenario: 顶级内容目录反映新栏目

- **WHEN** 开发者查看 `packages/vue-element-cui-nuxt/content/`
- **THEN** 内容目录必须围绕“快速开始”“组件”“规范”“更新”组织，而不是继续沿用旧页面分布

#### Scenario: 首页与栏目路径一一对应

- **WHEN** 用户从首页或一级导航进入任一栏目
- **THEN** 路由路径和内容目录必须与对应栏目保持一致

### Requirement: 组件文档必须按组件类型分组

系统 SHALL 按组件类型组织组件文档，并允许将同类旧页面合并为新的体系页。

#### Scenario: 组件栏目按类型而非业务场景展示

- **WHEN** 用户浏览“组件”栏目
- **THEN** 组件页面必须按数据展示、表单输入、选择器、反馈与弹层、导航与布局等类型组织

#### Scenario: 选择器类页面可以合并为体系页

- **WHEN** 旧文档中存在 `select`、`select-dict`、`select-enum`、`radio-enum` 等相近页面
- **THEN** 新结构必须允许把这些内容合并为统一的“选择器”体系页

### Requirement: 现有 markdown 内容必须尽量保留并重编排

系统 SHALL 在迁移过程中尽量复用现有 markdown 文案内容，优先重编排而不是整体重写。

#### Scenario: 旧文案整段可复用时直接迁入

- **WHEN** 某段旧文案仍然适用于新结构
- **THEN** 迁移过程应优先整段复用该内容

#### Scenario: 合并页面时按新结构拆段重组

- **WHEN** 多个旧页面被合并为一个体系页
- **THEN** 迁移过程必须允许把原有文案按“场景、演示、配置、注意事项”等新结构重排

### Requirement: 旧文档路径必须被正式废弃

系统 SHALL 删除旧文档路径作为正式入口，不保留旧路径兼容层。

#### Scenario: 旧路径不再出现在导航与正式入口中

- **WHEN** 用户浏览首页、导航或侧边栏
- **THEN** 不应再看到旧路径结构作为正式入口

#### Scenario: 新路径服务于新认知结构

- **WHEN** 用户访问新的组件或指南页面
- **THEN** 路径命名必须服务于新的类型分组和栏目结构，而不是为了兼容旧命名而保留历史包袱
