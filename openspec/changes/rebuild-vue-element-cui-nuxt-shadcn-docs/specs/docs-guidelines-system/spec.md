## ADDED Requirements

### Requirement: 文档站必须提供独立的规范栏目

系统 SHALL 提供独立的“规范”顶级栏目，用于承载跨组件共享的设计约定、最佳实践和开发约定。

#### Scenario: 规范栏目作为一级入口出现

- **WHEN** 用户查看首页入口或一级导航
- **THEN** “规范”必须作为独立顶级栏目出现，而不是隐藏在组件页内部

#### Scenario: 规范栏目具有独立内容路径

- **WHEN** 开发者查看内容目录
- **THEN** 规范内容必须位于独立栏目路径下，而不是散落在组件目录中

### Requirement: 规范栏目必须至少包含四类核心页面

系统 SHALL 在首轮重建中提供组件设计约定、表单与表格最佳实践、弹层与交互建议、开发约定四类核心页面。

#### Scenario: 组件设计约定页可被访问

- **WHEN** 用户进入规范栏目
- **THEN** 用户必须能够访问组件设计约定页面

#### Scenario: 开发约定页可被访问

- **WHEN** 用户进入规范栏目
- **THEN** 用户必须能够访问开发约定页面，并了解 demo 编写与文档组织规则

### Requirement: 规范内容必须抽离跨组件共性知识

系统 SHALL 将通用约定、最佳实践和交互建议沉淀到规范栏目中，减少组件页中的重复说明。

#### Scenario: 表单与表格共性经验被抽离

- **WHEN** 多个组件页都涉及相同的表单或表格实践
- **THEN** 这些共性内容必须优先沉淀到规范栏目，而不是在多个组件页重复散落

#### Scenario: 弹层与交互建议形成统一规则

- **WHEN** 文档涉及 dialog、dialog-form、search 等交互模式
- **THEN** 规范栏目必须提供统一的交互建议页来解释推荐模式和注意事项
