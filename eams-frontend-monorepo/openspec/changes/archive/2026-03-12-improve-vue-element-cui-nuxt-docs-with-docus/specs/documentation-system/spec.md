## MODIFIED Requirements

### 需求：Nuxt 3 项目结构

文档系统应当是一个位于 `packages/vue-element-cui-nuxt/` 的 Nuxt 3 项目，使用 Docus 框架，同时作为演练场和文档站点。

#### 场景：项目初始化

- **当** 开发者初始化 Nuxt 项目时
- **则** 项目应当包含 `content/`、`components/content/`、`app.config.ts` 目录和文件

#### 场景：开发服务器

- **当** 开发者运行 `pnpm dev` 时
- **则** Docus 开发服务器应当启动并支持热重载

#### 场景：移除 pages 目录

- **当** 开发者查看项目结构时
- **则** 不应存在 `pages/` 目录（Docus 使用基于内容的路由）

### 需求：Nuxt Content 集成

文档系统应当使用 Docus（基于 Nuxt Content）来实现基于 Markdown 的文档，并支持 MDC 语法和 Docus 扩展组件。

#### 场景：内容目录

- **当** 开发者创建文档时
- **则** Markdown 文件应当放置在 `content/` 根目录中，按照 Docus 标准结构组织

#### 场景：MDC 语法

- **当** 文档包含组件演示时
- **则** 应当支持使用 MDC 语法直接在 Markdown 中嵌入 Vue 组件

#### 场景：Docus 内置组件

- **当** 文档需要使用特殊组件时
- **则** 应当支持使用 Docus 内置的 MDC 组件（如 `::alert`、`::code-group` 等）

### 需求：文档结构

文档应当包含入门指南和各个组件的文档页面，采用 Docus 标准的目录结构。

#### 场景：首页

- **当** 用户访问文档根路径
- **则** 应当显示 `content/0.index.md` 定义的首页

#### 场景：入门页面

- **当** 用户访问文档时
- **则** 应当在 `content/1.getting-started/` 目录找到安装和快速入门指南

#### 场景：组件文档

- **当** 用户查看组件文档时
- **则** 每个组件应当在 `content/2.components/<component-name>.md` 有专用页面

#### 场景：自动路由生成

- **当** 开发者在 `content/` 目录创建 Markdown 文件时
- **则** Docus 应当自动生成对应的路由

### 需求：导航

文档应当提供完整的导航系统，包括顶部导航、侧边栏、面包屑和目录。

#### 场景：顶部导航

- **当** 用户查看文档时
- **则** 顶部应当显示在 `app.config.ts` 中配置的导航栏

#### 场景：侧边栏导航

- **当** 用户查看文档时
- **则** 侧边栏应当显示所有文档页面的层级导航（自动生成或手动配置）

#### 场景：当前页面指示

- **当** 用户在某个文档页面上时
- **则** 侧边栏应当高亮显示当前页面

#### 场景：面包屑导航

- **当** 用户查看文档页面时
- **则** 页面顶部应当显示面包屑导航

#### 场景：页面目录

- **当** 用户查看文档页面时
- **则** 页面右侧应当显示当前页面的目录（Table of Contents）

## REMOVED Requirements

### 需求：代码高亮

**Reason**: Docus 内置了更强大的代码高亮功能，不需要单独配置
**Migration**: 使用 Docus 的默认代码高亮配置，支持更多语言和主题

## ADDED Requirements

### 需求：Docus 配置文件

文档系统应当使用 `app.config.ts` 来配置 Docus 应用。

#### 场景：站点元信息配置

- **当** 开发者查看 `app.config.ts`
- **则** 应当包含站点名称、描述、URL、Logo 等元信息

#### 场景：导航配置

- **当** 开发者查看 `app.config.ts`
- **则** 应当包含顶部导航和侧边栏导航的配置

#### 场景：搜索配置

- **当** 开发者查看 `app.config.ts`
- **则** 应当包含搜索功能的配置

### 需求：深色模式

文档系统应当支持深色模式和浅色模式切换。

#### 场景：主题切换按钮

- **当** 用户访问文档站点
- **则** 顶部导航栏应当显示主题切换按钮

#### 场景：自动跟随系统主题

- **当** 用户未手动选择主题时
- **则** 应当自动跟随操作系统的主题设置

### 需求：搜索功能

文档系统应当提供搜索功能。

#### 场景：搜索按钮

- **当** 用户访问文档站点
- **则** 顶部导航栏应当显示搜索按钮

#### 场景：键盘快捷键

- **当** 用户按下 `Ctrl+K` 或 `Cmd+K`
- **则** 应当打开搜索对话框

#### 场景：搜索文档内容

- **当** 用户输入搜索关键词
- **则** 应当返回匹配的文档页面和内容摘要
