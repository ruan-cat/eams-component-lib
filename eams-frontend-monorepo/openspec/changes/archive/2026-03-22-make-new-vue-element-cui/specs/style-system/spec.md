## 新增需求

### 需求：SCSS 变量

样式系统应当定义一套全面的 SCSS 变量，用于颜色、排版、间距和其他设计令牌。

#### 场景：颜色变量

- **当** 样式被编译时
- **则** 应当包含主色、成功色、警告色、危险色和信息色的变量

#### 场景：排版变量

- **当** 样式被编译时
- **则** 应当包含字体大小的变量（基础、小号、大号）

#### 场景：间距变量

- **当** 样式被编译时
- **则** 应当包含间距比例的变量（xs、sm、md、lg、xl）

### 需求：BEM 命名规范

样式系统应当对所有 CSS 类使用 BEM（块元素修饰符）命名规范。

#### 场景：块命名

- **当** 组件定义根类时
- **则** 应当使用格式 `.cui-<组件名>`

#### 场景：元素命名

- **当** 组件定义子元素类时
- **则** 应当使用格式 `.cui-<组件名>__<元素>`

#### 场景：修饰符命名

- **当** 组件定义变体类时
- **则** 应当使用格式 `.cui-<组件名>--<修饰符>`

### 需求：SCSS 混入

样式系统应当提供可复用的 SCSS 混入，用于常见模式。

#### 场景：BEM 混入

- **当** 开发者编写组件样式时
- **则** 应当使用 `@mixin b($block)`、`@mixin e($element)`、`@mixin m($modifier)` 来构建 BEM 结构

#### 场景：工具混入

- **当** 开发者需要常见模式时
- **则** 应当提供用于清除浮动、文本省略和其他工具的混入

### 需求：组件样式

每个组件应当有自己的 SCSS 文件，可以独立导入。

#### 场景：组件样式文件

- **当** 组件被创建时
- **则** 应当在 `src/styles/components/` 中包含对应的 SCSS 文件

#### 场景：独立导入

- **当** 用户只想导入特定组件样式时
- **则** 应当能够导入 `vue-element-cui/styles/<组件名>.css`

### 需求：主样式入口

样式系统应当提供一个主入口点，导入所有组件样式。

#### 场景：完整导入

- **当** 用户想要所有组件样式时
- **则** 应当导入 `vue-element-cui/styles/index.css`

#### 场景：样式编译

- **当** 构建运行时
- **则** 主 SCSS 文件应当编译为包含所有样式的单个 CSS 文件

### 需求：主题定制

样式系统应当通过 CSS 变量支持主题定制。

#### 场景：CSS 变量导出

- **当** 样式被编译时
- **则** 设计令牌应当作为 CSS 自定义属性可用

#### 场景：变量覆盖

- **当** 用户想要定制主题时
- **则** 应当能够在自己的样式表中覆盖 CSS 变量
