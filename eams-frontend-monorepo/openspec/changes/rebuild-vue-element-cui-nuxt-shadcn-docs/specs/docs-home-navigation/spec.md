## ADDED Requirements

### Requirement: 首页必须作为组件库入口页工作

系统 SHALL 将文档站首页设计为组件库入口页，优先提供安装、快速开始、组件分类和规范入口，而不是品牌营销型首页。

#### Scenario: 首页提供快速进入关键栏目

- **WHEN** 用户访问文档站首页
- **THEN** 页面必须提供进入“快速开始”“组件”“规范”“更新”等关键栏目的明确入口

#### Scenario: 首页亮点不替代主入口

- **WHEN** 首页展示组件亮点或示例
- **THEN** 这些内容必须服务于文档入口，而不能取代安装、导航和分类入口的主地位

### Requirement: 顶级导航必须固定为四个一级栏目

系统 SHALL 提供稳定的一级导航结构，并至少包含“快速开始”“组件”“规范”“更新”四个顶级栏目。

#### Scenario: 顶部导航展示四个一级栏目

- **WHEN** 用户查看文档站顶部导航
- **THEN** 顶部导航必须包含“快速开始”“组件”“规范”“更新”

#### Scenario: 一级栏目与内容目录保持一致

- **WHEN** 用户通过导航进入任一顶级栏目
- **THEN** 对应内容目录和页面结构必须与该导航栏目一致，不应出现旧路径主导的混乱入口

### Requirement: 导航体验必须同时覆盖桌面端和移动端

系统 SHALL 提供可用的桌面端与移动端导航体验，保证组件库文档在两种场景下均可正常浏览。

#### Scenario: 桌面端可以访问主导航和侧边导航

- **WHEN** 用户在桌面端浏览文档站
- **THEN** 用户必须能够稳定使用顶部导航、侧边导航和页面目录

#### Scenario: 移动端可以展开并访问导航入口

- **WHEN** 用户在移动端浏览文档站
- **THEN** 用户必须能够展开移动端导航并访问各一级栏目和当前内容结构
