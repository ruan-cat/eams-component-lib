# 选择器体系总览

选择器体系重点处理“候选项从哪里来、用户怎么选、是否需要确认”这三类问题。

## 选型建议

1. 普通单选、多选场景优先使用 `CuiSelect`。
2. 当选项来自后端字典或稳定枚举时，优先使用字典、枚举专用组件。
3. 当需要组合搜索、确认或大批量候选项时，再考虑 `CuiSelectBox` 或弹层式选择。

## 组件概览

| 组件 | 适用场景 |
| --- | --- |
| `CuiSelect` | 普通静态选项、远程搜索 |
| `CuiSelectEnum` | 选项来自枚举服务 |
| `CuiSelectDict` | 选项来自字典服务 |
| `CuiSelectBox` | 弹层式复杂选择、需要二次确认 |
| `CuiRadioEnum` | 枚举数据的单选按钮组 |
| `CuiRadioDict` | 字典数据的单选按钮组 |
| `CuiCheckboxEnum` | 枚举数据的多选按钮组 |
